🏛️ Sistem Mimarisi & Canli Veri Akisi Hexagonal · DDD · CQRS
①②③ Veri yukari: ESP32 → Broker → Backend (siradaki kutu parlar) ④ Backend → PostgreSQL kaydi ❶❷❸ Komut asagi: Panel → Backend → Broker → ESP32
  • Hexagonal (Ports & Adapters): Cekirdek domain dis dunyadan izole; MQTT, HTTP, WebSocket ve PostgreSQL birer adapter olarak port arayuzlerine baglanir.
  • DDD: Cabin ve User aggregate'leri, Thresholds/DecisionConfig value object'leri ile invariant'lar (orn. min<max) domain katmaninda zorlanir.
  • CQRS-vari ayrim: Yuksek frekansli telemetri Cabin aggregate'inden gecmez; ayri ReadingStore yazma yolu + read-model sorgusu kullanilir.
  • Otorite cihazda: Aktuator durumunun tek dogru kaynagi ESP32'nin up/state mesajidir; backend komutu iyimser guncellemez. Internet kopsa kabin otonom karar verir.
